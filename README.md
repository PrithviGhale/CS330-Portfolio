CS 330 – Computational Graphics and Visualization
Portfolio Reflection – Final Project

How do I approach designing software?

When I design software, I first try to understand the goal of the project and what the final result should look like. For my 3D scene, I focused on how I wanted the environment to look and how objects should be placed. I think about the structure first before jumping into coding. I try to break the design into smaller parts like objects, lighting, textures, and camera setup.

What new design skills did this project help me develop?

This project helped me improve my understanding of 3D object placement, lighting, and camera positioning. I learned how small design choices, like texture selection and light placement, can completely change how a scene looks. I also became better at organizing objects logically inside the scene.

What design process did I follow?

I followed an iterative design process. I started simple with basic shapes, then slowly added textures, lighting, and camera controls. After each change, I tested the scene and adjusted it. This helped me improve the scene step by step instead of trying to make everything perfect at once.

How could this design approach be applied in future work?

This approach can be used in future software projects by starting small, testing often, and improving gradually. It helps prevent large errors and makes development more manageable.

How do I approach developing programs?

When developing programs, I focus on writing clear and organized code. I try to separate different parts of the logic into functions so the program is easier to understand and debug.

What new development strategies did I use?

I used modular design and focused more on debugging through testing small parts of code at a time. I also improved my understanding of OpenGL functions and how rendering pipelines work.

How did iteration factor into development?

Iteration was very important. Each milestone helped me improve the project. I would test lighting, adjust textures, fix camera issues, and improve movement step by step.

How has my coding approach evolved?

At the beginning of the course, I was mainly focused on making the code work. By the end, I was more focused on structure, organization, and readability. I also became more comfortable debugging graphical errors.

How can computer science help me reach my goals?

Computer science gives me the technical skills needed to build real applications. It helps me think logically and solve complex problems. These skills are important for my future career in software development.

How do computational graphics and visualizations help in my educational pathway?

They help me understand how graphics systems work, including rendering, lighting, and 3D transformations. This builds a strong foundation for advanced graphics or game development courses.

How do computational graphics and visualizations help in my professional pathway?

In a professional setting, these skills can be applied in game development, simulations, UI design, and visualization systems. Understanding graphics pipelines and 3D environments makes me more versatile as a developer.
